TCP IP与OSI模型

1 OSI 七层模型

Open System Interconnection Model
一个开放性的参考模型
核心思想：

  1. 通信两端的主机能够理解对方的语言
  2. 希望通信系统模块化，每个模块提供标准接口

具体每一层的作用

  • 应用层：应用程序与网络之间的接口：如http，ftp
  • 表示层：数据格式的转换
  • 会话层：建立，管理和维护通信
  • 传输层：两个应用程序之间的数据传输，如TCP，UDP
  • 网络层：寻找和路由，将数据传输到目标地址 如TP ICMP
  • 链路层：互联设备之间的传输，数据帧转换为比特流
  • 物理层：比特流转换为电子信号

2 TCP/IP 模型

TCP/IP并不是特指TCP与IP两种协议，然而在很多情况下，它只是利用 IP 进行通信时所必须用到的协议群的统称。具体来说，IP 或 ICMP、TCP 或 UDP、TELNET 或 FTP、以及 HTTP 等都属于 TCP/IP 协议。他们与 TCP 或 IP 的关系紧密，是互联网必不可少的组成部分。TCP/IP 一词泛指这些协议，因此，有时也称 TCP/IP 为网际协议群。

对应的模型

  • 应用层：HTTP，SMTP
  • 传输层：TCP，UDP，UGP
  • 网络层：IP，ICMP，IGMP
  • 链路层：ARP,RARP

区别与不同


OSI 参考模型注重“通信协议必要的功能是什么”，价值在于用户它的分层和协议思想，先定义后实现
TCP/IP协议更强调“在计算机上实现协议应该开发哪种程序”，价值在于实际应用，现有实现后有模型定义。
